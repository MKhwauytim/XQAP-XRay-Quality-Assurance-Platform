# Desktop App-Mode Shortcut Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a Windows app-mode desktop shortcut (icon + PowerShell creation script + .bat wrapper) that gives the existing single-file build an installed-app look — its own window, no tabs/address bar — with zero changes to the build pipeline itself.

**Architecture:** Three static files live under `public/`, which Vite already copies verbatim into `dist/` on every build with no config change. A one-time, dev-only PowerShell script generates the icon asset (committed to git, never regenerated by CI). The shipped PowerShell script self-locates, detects an installed Chromium browser, and creates a re-runnable Desktop `.lnk` shortcut via the `WScript.Shell` COM object.

**Tech Stack:** PowerShell 7 (`pwsh`) + .NET `System.Drawing` (GDI+, confirmed available in this environment) for icon generation; Windows Script Host COM (`WScript.Shell`) for shortcut creation — no new npm dependencies.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-16-desktop-shortcut-design.md` — read for full rationale.
- No new npm dependencies; `npm run build` / `npm run dev` / `npm run preview` stay unchanged.
- No service worker, no web manifest, no bundled local server, no second build output folder.
- All new files under `public/` are shipped (copied into `dist/` verbatim by Vite); `scripts/generate-app-icon.ps1` is dev-only tooling and is NOT under `public/`.
- Shortcut name (exact): `نظام معالجة بيانات الأشعة.lnk` on the user's Desktop.
- Browser detection order: Chrome first (three standard install paths), then Edge (two standard install paths) as the universal fallback.
- The `--app=` argument's `file:///` URI must survive as one token through spaces and parentheses in the deployment path (this repo's own path already contains both) — verify this explicitly, don't assume it.
- This feature has no Vitest-testable logic (it's PowerShell/Windows tooling, not application TypeScript). Verification is running the real scripts against real files/paths and inspecting real output — every task below specifies the exact command and exact expected result instead of a `pytest`/`vitest` step.
- EDIT_LOG (CLAUDE.md): every task records its files under a new `## v55.1 — 2026-07-16 — Desktop app-mode shortcut` entry (or the next available decimal bump — check `docs/EDIT_LOG.md`'s current top entry before writing the version number) in `docs/EDIT_LOG.md`, added BEFORE the code/asset changes, with Before/After per file.
- Working directory for all commands: `C:\Users\WorkNStudy\Desktop\New folder (2)\x-ray-quality-app-v1` — note this path itself contains a space and parentheses, which doubles as a live test case for the quoting requirement above.

- [ ] **Step 2: Write the icon generator script**

Create `scripts/generate-app-icon.ps1`:

```powershell
# One-time generator for public/app-icon.ico. Reproduces the shapes of the
# inline SVG favicon in index.html (navy rounded-square mark, white "film"
# panel, three scan lines, teal checkmark badge) at 16/32/48/256px using
# GDI+ primitives, then hand-assembles a real multi-resolution Windows ICO
# with PNG-compressed frames. Re-run only if the brand mark changes — the
# output is committed to git, not regenerated by any build step.

$ErrorActionPreference = "Stop"
Add-Type -AssemblyName System.Drawing

$outputPath = Join-Path $PSScriptRoot "..\public\app-icon.ico"
$sizes = @(16, 32, 48, 256)

function New-RoundedRectPath {
    param([float]$X, [float]$Y, [float]$W, [float]$H, [float]$R)
    $path = New-Object System.Drawing.Drawing2D.GraphicsPath
    $d = $R * 2
    $path.AddArc($X, $Y, $d, $d, 180, 90)
    $path.AddArc($X + $W - $d, $Y, $d, $d, 270, 90)
    $path.AddArc($X + $W - $d, $Y + $H - $d, $d, $d, 0, 90)
    $path.AddArc($X, $Y + $H - $d, $d, $d, 90, 90)
    $path.CloseFigure()
    return $path
}

function New-IconFrame {
    param([int]$Size)

    $bmp = New-Object System.Drawing.Bitmap($Size, $Size)
    $g = [System.Drawing.Graphics]::FromImage($bmp)
    $g.SmoothingMode = [System.Drawing.Drawing2D.SmoothingMode]::AntiAlias
    $g.Clear([System.Drawing.Color]::Transparent)

    $scale = $Size / 96.0
    $navy  = [System.Drawing.Color]::FromArgb(255, 0x17, 0x36, 0x5D)
    $film  = [System.Drawing.Color]::FromArgb(255, 0xF8, 0xFB, 0xFF)
    $teal  = [System.Drawing.Color]::FromArgb(255, 0x00, 0xA3, 0xA3)
    $white = [System.Drawing.Color]::White

    $bgPath = New-RoundedRectPath -X 0 -Y 0 -W $Size -H $Size -R (18 * $scale)
    $g.FillPath((New-Object System.Drawing.SolidBrush($navy)), $bgPath)

    $filmPath = New-RoundedRectPath -X (32 * $scale) -Y (30 * $scale) -W (32 * $scale) -H (28 * $scale) -R (8 * $scale)
    $g.FillPath((New-Object System.Drawing.SolidBrush($film)), $filmPath)

    if ($Size -ge 32) {
        $pen = New-Object System.Drawing.Pen($navy, [Math]::Max(1.5, 5 * $scale))
        $pen.StartCap = [System.Drawing.Drawing2D.LineCap]::Round
        $pen.EndCap = [System.Drawing.Drawing2D.LineCap]::Round
        $g.DrawLine($pen, (36 * $scale), (38 * $scale), (60 * $scale), (38 * $scale))
        $g.DrawLine($pen, (36 * $scale), (48 * $scale), (68 * $scale), (48 * $scale))
        $g.DrawLine($pen, (36 * $scale), (58 * $scale), (56 * $scale), (58 * $scale))
    }

    $badgeR = 10 * $scale
    $badgeCx = 66 * $scale
    $badgeCy = 60 * $scale
    $g.FillEllipse((New-Object System.Drawing.SolidBrush($teal)), ($badgeCx - $badgeR), ($badgeCy - $badgeR), ($badgeR * 2), ($badgeR * 2))

    if ($Size -ge 32) {
        $checkPen = New-Object System.Drawing.Pen($white, [Math]::Max(1, 3 * $scale))
        $checkPen.StartCap = [System.Drawing.Drawing2D.LineCap]::Round
        $checkPen.EndCap = [System.Drawing.Drawing2D.LineCap]::Round
        $checkPen.LineJoin = [System.Drawing.Drawing2D.LineJoin]::Round
        $pts = @(
            (New-Object System.Drawing.PointF((62 * $scale), (60 * $scale))),
            (New-Object System.Drawing.PointF((65 * $scale), (63 * $scale))),
            (New-Object System.Drawing.PointF((71 * $scale), (56 * $scale)))
        )
        $g.DrawLines($checkPen, $pts)
    }

    $g.Dispose()
    return $bmp
}

function ConvertTo-PngBytes {
    param([System.Drawing.Bitmap]$Bitmap)
    $ms = New-Object System.IO.MemoryStream
    $Bitmap.Save($ms, [System.Drawing.Imaging.ImageFormat]::Png)
    return $ms.ToArray()
}

$frames = foreach ($size in $sizes) {
    $bmp = New-IconFrame -Size $size
    $png = ConvertTo-PngBytes -Bitmap $bmp
    $bmp.Dispose()
    [pscustomobject]@{ Size = $size; Png = $png }
}

$stream = New-Object System.IO.MemoryStream
$writer = New-Object System.IO.BinaryWriter($stream)

$writer.Write([UInt16]0)
$writer.Write([UInt16]1)
$writer.Write([UInt16]$frames.Count)

$headerSize = 6 + (16 * $frames.Count)
$offset = $headerSize
foreach ($frame in $frames) {
    $dim = if ($frame.Size -ge 256) { 0 } else { [byte]$frame.Size }
    $writer.Write([byte]$dim)
    $writer.Write([byte]$dim)
    $writer.Write([byte]0)
    $writer.Write([byte]0)
    $writer.Write([UInt16]1)
    $writer.Write([UInt16]32)
    $writer.Write([UInt32]$frame.Png.Length)
    $writer.Write([UInt32]$offset)
    $offset += $frame.Png.Length
}
foreach ($frame in $frames) {
    $writer.Write($frame.Png)
}

$writer.Flush()
[System.IO.File]::WriteAllBytes($outputPath, $stream.ToArray())
$writer.Dispose()
$stream.Dispose()

Write-Host "Wrote $outputPath ($($frames.Count) frames: $($sizes -join ', ')px)" -ForegroundColor Green
```

- [ ] **Step 2: Write the script**

Create `public/create-desktop-shortcut.ps1`:

```powershell
# Creates a Desktop shortcut that launches this app in browser "app mode"
# (its own window, no tabs/address bar) via Chrome or Edge, using the icon
# shipped alongside this script. Safe to re-run — e.g. after moving this
# folder to a new location, re-running repairs the shortcut's target path.

$ErrorActionPreference = "Stop"

$scriptDir = Split-Path -Parent $MyInvocation.MyCommand.Path
$indexPath = Join-Path $scriptDir "index.html"
$iconPath  = Join-Path $scriptDir "app-icon.ico"

if (-not (Test-Path $indexPath)) {
    Write-Host "خطأ: لم يتم العثور على index.html في نفس مجلد هذا السكربت." -ForegroundColor Red
    Write-Host "تأكد من تشغيل هذا الملف من داخل مجلد التطبيق دون نقله بمفرده." -ForegroundColor Red
    exit 1
}

$browserCandidates = @(
    "$env:ProgramFiles\Google\Chrome\Application\chrome.exe",
    "${env:ProgramFiles(x86)}\Google\Chrome\Application\chrome.exe",
    "$env:LocalAppData\Google\Chrome\Application\chrome.exe",
    "$env:ProgramFiles\Microsoft\Edge\Application\msedge.exe",
    "${env:ProgramFiles(x86)}\Microsoft\Edge\Application\msedge.exe"
)

$browserPath = $browserCandidates | Where-Object { Test-Path $_ } | Select-Object -First 1

if (-not $browserPath) {
    Write-Host "خطأ: تعذر العثور على متصفح Chrome أو Edge على هذا الجهاز." -ForegroundColor Red
    Write-Host "يتطلب هذا التطبيق أحد المتصفحين لتفعيل الوصول إلى مجلد العمل." -ForegroundColor Red
    exit 1
}

$resolvedIndex = (Resolve-Path $indexPath).Path
$fileUri = "file:///" + ($resolvedIndex -replace '\\', '/')

$desktop = [Environment]::GetFolderPath("Desktop")
$shortcutName = "نظام معالجة بيانات الأشعة.lnk"
$shortcutPath = Join-Path $desktop $shortcutName

$shell = New-Object -ComObject WScript.Shell
$shortcut = $shell.CreateShortcut($shortcutPath)
$shortcut.TargetPath = $browserPath
$shortcut.Arguments = "--app=`"$fileUri`" --new-window"
$shortcut.WorkingDirectory = $scriptDir
$shortcut.Description = "نظام معالجة بيانات الأشعة"
if (Test-Path $iconPath) {
    $shortcut.IconLocation = (Resolve-Path $iconPath).Path
}
$shortcut.Save()

Write-Host "تم إنشاء اختصار على سطح المكتب: $shortcutName" -ForegroundColor Green
```


(`%~dp0` expands to this `.bat` file's own directory with a trailing backslash, so `%~dp0create-desktop-shortcut.ps1` resolves correctly regardless of where the folder is copied to. `pause` keeps the console window open after the script finishes so the user can read the success/error message before it closes — otherwise a double-clicked `.bat` closes its window immediately on completion.)
