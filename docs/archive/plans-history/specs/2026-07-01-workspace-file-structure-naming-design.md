# Workspace File-Structure Naming — Design

**Date:** 2026-07-01
**Status:** Approved (pending spec review)
**Context:** Greenfield — no production workspaces on disk must be preserved. Renames can be applied freely; only tests and docs need to follow.

## Problem

The folder/file names the app generates on disk are inconsistent, and the names
themselves are scattered across many modules (some as local `const`s, some as
hardcoded string literals). Three separate conventions coexist today:

- **Roots:** `1-Population`, `2-Samples`, `4-Reports`, `5-System`, `6-Templates`
  are `N-Titlecaseword`, but `3-User Data` has a space and two capitalized words.
- **Subfolders:** numbered title-case (`1-Main`, `2-Employees`, `3-Approvals`,
  `2-Audit`, `3-Backups`, `1-Locks`) mixed with unnumbered lowercase
  (`raw`, `processed`, `designs`, `powerbi-export`, `user-presets`, `feedback`).
- **Files:** dot-namespaced (`population.final.json`) alongside hyphenated
  (`inspection-template-selection.json`) and one French-named `LISEZMOI.txt`.

Because the literals are scattered (`ACTIVITY_AUDIT_FOLDER` in `authActivityLog.ts`,
`USER_PRESETS_FOLDER` in `browsePresetStorage.ts`, hardcoded `"powerbi-export"`,
`"designs"`, `"1-Main"` in various modules, plus a *second* numbered set
`1-Locks`/`2-Audit`/`3-Backups` in `WORKSPACE_FILE_NAMES`), the inconsistency is
structural: there is no single place that defines the tree, so it drifts.

## Goals

1. One naming convention for the entire generated tree.
2. One source of truth for every folder/file name, so consistency is enforced by
   construction and a future rename is a one-line change.

Non-goals: no data migration (greenfield), no reorganization of *what data lives
where*, no merging of the parallel `workspaceDefaults` flat scheme with the
population/sampling storage scheme (both already dot-namespaced). No changes to
browser storage keys (`xray_*`).

## The convention (three rules)

1. **Casing** — everything lowercase; multi-word tokens use hyphens
   (`kebab-case`). No spaces, no title case.
2. **Numbering** — an `N-` prefix *only* when sibling folders form an ordered
   workflow the user moves **through**. Independent buckets get no number.
3. **Files** — `entity.qualifier[.sub].json`: dot-separated namespace segments,
   kebab inside a segment. One entity per file family.

### Numbering decisions (rule 2 applied)

| Folder group | Numbered? | Why |
|---|---|---|
| The 6 roots | ✅ | They are the top-level workflow index (population → samples → …). |
| Samples month: main → employees → approvals | ✅ | A real sequence: draw → distribute → approve. |
| Population month: raw → processed | ✅ | A real 2-step pipeline (data lineage). Siblings under the month folder, not nested. |
| System: locks, audit, backups, powerbi-export, user-presets, feedback | ❌ | Independent peer buckets — no order to move through. |
| Reports: designs | ❌ | Single bucket. |

## Structural fix — single source of truth

Promote every folder/file name into [`workspacePaths.ts`](../../../src/data/workspace/workspacePaths.ts).
`WORKSPACE_ROOTS` already exists; add:

```ts
export const POPULATION_SUBFOLDERS = { raw: "1-raw", processed: "2-processed" } as const;
export const SAMPLE_SUBFOLDERS   = { main: "1-main", employees: "2-employees", approvals: "3-approvals" } as const;
export const SYSTEM_SUBFOLDERS   = {
  locks: "locks", audit: "audit", backups: "backups",
  powerbiExport: "powerbi-export", userPresets: "user-presets", feedback: "feedback",
} as const;
export const REPORTS_SUBFOLDERS  = { designs: "designs" } as const;
```

Then:

- Refactor the getter functions in `workspacePaths.ts` (`getSampleMainDir`,
  `getSampleEmployeeDir`, `getSampleApprovalsDir`) to use `SAMPLE_SUBFOLDERS`
  instead of the inline `"1-Main"` / `"2-Employees"` / `"3-Approvals"` literals.
- Delete the scattered local constants and hardcoded literals, importing the
  central names instead:
  - `authActivityLog.ts` — `ACTIVITY_AUDIT_FOLDER` → `SYSTEM_SUBFOLDERS.audit`
  - `browsePresetStorage.ts` — `USER_PRESETS_FOLDER` → `SYSTEM_SUBFOLDERS.userPresets`
  - `feedbackStorage.ts` — `FEEDBACK_FOLDER` → `SYSTEM_SUBFOLDERS.feedback`
  - `exportWriter.ts` — `"powerbi-export"` → `SYSTEM_SUBFOLDERS.powerbiExport`
  - `reportDesignStorage.ts` — `"designs"` → `REPORTS_SUBFOLDERS.designs`
  - `populationStorage.ts` / `sampleStorage.ts` — `"raw"` / `"processed"` →
    `POPULATION_SUBFOLDERS.*`
  - `backupStorage.ts` — backups subfolder → `SYSTEM_SUBFOLDERS.backups`
- Reconcile `WORKSPACE_FILE_NAMES` in `workspaceDefaults.ts`: replace
  `locksFolder: "1-Locks"`, `auditFolder: "2-Audit"`, `backupsFolder: "3-Backups"`
  with references to `SYSTEM_SUBFOLDERS`, and the `employeeAnswersFolder` string
  `${WORKSPACE_ROOTS.samples}/{month}/2-Employees` with `SAMPLE_SUBFOLDERS.employees`.
- `fileSystemAccess.ts` — replace hardcoded `"1-Population"`, `"3-User Data"`,
  `"4-Reports"` in the workspace-init path with `WORKSPACE_ROOTS.*`.

After this pass, no folder/file name string exists outside the constant modules
(except tests, which assert on concrete names).

## Legacy fallbacks

The existing `LEGACY_ROOTS` fallback (`Population/`, `.system/`, `templates/`)
and the legacy population-month `sample` / `employee-answers` / `approvals`
fallbacks are left untouched — harmless, and greenfield means they won't be hit.
The old title-case current names (`1-Population`, etc.) are **not** added as
fallbacks (greenfield; no such data exists).

## Testing

- Update every `*.test.ts` that hardcodes an old name (`fileSystemAccess.test.ts`,
  `populationStorage.test.ts`, `templateStorage.test.ts`, `exportWriter.test.ts`,
  `authActivityLog.test.ts`, and any others surfaced by a repo-wide grep) to the
  new names.
- Add one focused assertion (or extend an existing structure test) that the
  workspace-init path creates the exact target tree above.
- Run `npm run test:run`, `npm run lint`, and `npm run build` — the build must
  still produce a single `dist/index.html` with no broken references.
