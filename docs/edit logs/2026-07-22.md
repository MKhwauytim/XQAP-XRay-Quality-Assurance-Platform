## v59.1 — 2026-07-22 — Fix: slideshow scroll-restore firing on load + print-toggle click conflict

Task-review fix for two Important findings against v59.0's `DECK_FULLSCREEN_SCRIPT`.

**Bug 1 — unconditional scroll-restore on page load.** `sync()` is called once, unconditionally,
at the end of the script's IIFE, on every page load (not only after a real fullscreen exit).
Because `current()` is falsy before the user ever enters fullscreen, that initial call fell into
`sync()`'s `else` branch, which unconditionally ran `slides[activeIndex].scrollIntoView({ block:
'start' })`. With `activeIndex` starting at `0`, every fresh page load silently scrolled to slide 0
(the cover) — overriding the browser's native URL-fragment scroll, e.g. a link to
`#slide-recommendations` from the report's own side-nav (`DECK_NAV_SCRIPT`). Fixed with an
`everActivated` flag, set only once a real fullscreen session has been entered (inside `sync()`'s
`if (active)` branch, alongside where `activeIndex` gets recomputed from viewport geometry); the
`else` branch's `scrollIntoView` restoration now only runs when `everActivated` is true, so it fires
on a genuine "just exited fullscreen" transition and never on initial load.

**Bug 2 — click-to-advance conflicts with the per-slide print-include toggle.** Every slide already
renders a print-include checkbox (pre-existing feature, unrelated to this task) as a direct child of
`.slide`, wrapped in a `<div class="slide-controls">` (`slideControls()` in `slides.ts`, styled in
`theme.ts`). The new document-level click-to-advance handler's exclusion list
(`.btn-slide-nav, .deck-slide-counter, #deck-fullscreen-button`) didn't cover it, so clicking that
toggle during a slideshow both flipped print-inclusion state and advanced the slide. Confirmed the
real class from `slides.ts`/`theme.ts` (`.slide-controls` is the positioned wrapper `slideControls()`
renders around `printToggle()`'s `.slide-print-toggle` label, and, in dev-preview builds, the
variant switcher) and added it to the `closest()` exclusion list.

## v59.0 — 2026-07-22 — Add: true slideshow presentation mode for deck2 fullscreen

Task 3 of 3 (final) of the deck2 slideshow-mode plan — the point where the behavior actually goes
live. Task 1 added the `expand`/`compress` icons; Task 2 added the single-slide-presentation CSS
contract (`.deck-slide-active`, `#deck-slide-prev`/`#deck-slide-next`/`#deck-slide-counter`,
`body.deck-controls-visible`, `.btn-fullscreen-icon-expand`/`.btn-fullscreen-icon-compress`); this
task wires it up. `DECK_FULLSCREEN_SCRIPT` now tracks an `activeIndex` into `.slide` elements and
exposes `goTo`/`renderSlide`/`showControls` helpers: clicking/arrow-keying/space/page-up/page-down
moves exactly one slide, a fading prev/next/counter cluster (auto-hides after 2.5s idle, same as the
fullscreen button already did) appears via `body.deck-controls-visible`, and clicking anywhere on the
slide (excluding the nav buttons, counter, and the fullscreen button itself) advances to the next
slide. Entering fullscreen now computes the starting slide index from `getBoundingClientRect()`
*before* toggling the `deck-fullscreen` class, so it reads normal (not collapsed) layout geometry.
The fullscreen button itself switched from a text label (`ملء الشاشة` / `إنهاء ملء الشاشة`) to an
icon-only expand/compress toggle (Task 2's CSS shows/hides the two icon spans based on
`aria-pressed`), and the prev/next buttons get two new configurable Arabic labels
(`exec_deck_slideshow_prev`/`exec_deck_slideshow_next`) registered in the Settings label-override list
alongside the existing fullscreen-label pair.

  var root = document.documentElement;
  var request = root.requestFullscreen || root.webkitRequestFullscreen;
  var exit = document.exitFullscreen || document.webkitExitFullscreen;
  function current(){ return document.fullscreenElement || document.webkitFullscreenElement; }
  function disable(){
    document.body.classList.remove('deck-fullscreen');
    button.hidden = true;
  }
  if (typeof request !== 'function' || typeof exit !== 'function') { disable(); return; }

  var slides = Array.prototype.slice.call(document.querySelectorAll('.slide'));
  var activeIndex = 0;
  var prevBtn = document.getElementById('deck-slide-prev');
  var nextBtn = document.getElementById('deck-slide-next');
  var counter = document.getElementById('deck-slide-counter');
  var hideTimer = null;

  function showControls(){
    document.body.classList.add('deck-controls-visible');
    if (hideTimer) clearTimeout(hideTimer);
    hideTimer = setTimeout(function(){ document.body.classList.remove('deck-controls-visible'); }, 2500);
  }

  function renderSlide(){
    for (var i = 0; i < slides.length; i++) {
      slides[i].classList.toggle('deck-slide-active', i === activeIndex);
    }
    if (counter) counter.textContent = (activeIndex + 1) + ' / ' + slides.length;
    if (prevBtn) prevBtn.disabled = activeIndex === 0;
    if (nextBtn) nextBtn.disabled = activeIndex === slides.length - 1;
  }

  function goTo(index){
    if (index < 0 || index >= slides.length || index === activeIndex) return;
    activeIndex = index;
    renderSlide();
  }

  function sync(){
    var active = Boolean(current());
    var label = button.getAttribute(active ? 'data-exit-label' : 'data-enter-label');
    if (active) {
      var thresholdY = window.innerHeight * 0.35;
      var idx = 0;
      for (var i = 0; i < slides.length; i++) {
        if (slides[i].getBoundingClientRect().top <= thresholdY) idx = i; else break;
      }
      activeIndex = idx;
    }
    document.body.classList.toggle('deck-fullscreen', active);
    button.setAttribute('aria-pressed', active ? 'true' : 'false');
    button.setAttribute('aria-label', label);
    button.setAttribute('title', label);
    if (active) {
      renderSlide();
      showControls();
    } else {
      document.body.classList.remove('deck-controls-visible');
      if (hideTimer) clearTimeout(hideTimer);
      var el = slides[activeIndex];
      if (el && el.scrollIntoView) el.scrollIntoView({ block: 'start' });
    }
  }

  button.addEventListener('click', function(){
    var action;
    try { action = current() ? exit.call(document) : request.call(root); }
    catch (_) { disable(); return; }
    if (action && typeof action.catch === 'function') action.catch(disable);
  });
  document.addEventListener('fullscreenchange', sync);
  document.addEventListener('webkitfullscreenchange', sync);
  document.addEventListener('fullscreenerror', disable);
  document.addEventListener('webkitfullscreenerror', disable);

  if (prevBtn) prevBtn.addEventListener('click', function(e){ e.stopPropagation(); goTo(activeIndex - 1); showControls(); });
  if (nextBtn) nextBtn.addEventListener('click', function(e){ e.stopPropagation(); goTo(activeIndex + 1); showControls(); });

  document.addEventListener('keydown', function(e){
    if (!current()) return;
    if (e.key === 'ArrowLeft' || e.key === 'PageUp') { goTo(activeIndex - 1); showControls(); }
    else if (e.key === 'ArrowRight' || e.key === ' ' || e.key === 'PageDown') { goTo(activeIndex + 1); showControls(); }

  document.addEventListener('click', function(e){
    if (!current()) return;
    if (e.target.closest('.btn-slide-nav, .deck-slide-counter, #deck-fullscreen-button')) return;
    goTo(activeIndex + 1);
    showControls();
  });

  document.addEventListener('mousemove', function(){ if (current()) showControls(); });

  sync();
})();`;

## v58.3 — 2026-07-22 — Fix: fullscreen button was structurally hidden by its own toolbar CSS

Task-review fix for v58.2's fullscreen CSS. `body.deck-fullscreen .deck-toolbar>*{display:none;}`
was written assuming `#deck-fullscreen-button.btn-fullscreen` is a direct child of `.deck-toolbar`,
but the real markup (`index.ts`) nests it inside a `.deck-toolbar-actions` wrapper div alongside the
theme-toggle label and the print button — `.deck-toolbar-actions` is the actual direct child. The
`>*` selector therefore matched and hid `.deck-toolbar-actions` itself, taking the fullscreen button
down with it via `display:none` on an ancestor — a state no descendant rule can undo, so the
following `.deck-toolbar .btn-fullscreen{display:inline-flex;...}` override was dead code. Net
effect: the button that is supposed to fade in on mousemove during slideshow mode never rendered at
all. Fix targets the real direct child (`.deck-toolbar>.deck-brand`) plus everything inside the
actions wrapper except the fullscreen button itself
(`.deck-toolbar-actions>*:not(.btn-fullscreen)`), and gives `.btn-fullscreen` its own unconditional
fixed-position/fade rule no longer nested under an ancestor that could re-hide it.

## v58.2 — 2026-07-22 — Change: fullscreen CSS from scroll-stack to single-slide layout

Replaces what `body.deck-fullscreen` means for the executive report's (deck2) fullscreen button:
previously a scrollable, scroll-snapped stack of all slides with a toolbar pinned near the top;
now the CSS contract for a true one-slide-at-a-time presentation mode, where exactly one `.slide`
(marked `.deck-slide-active`) fills the viewport at a time. This is CSS only — Task 2 of a 3-task
slideshow-mode plan. The class names/selectors this stylesheet now targets

the completed JSON backup is still recorded successfully. Existing backup manifests retain their
